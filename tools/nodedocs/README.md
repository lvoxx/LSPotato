# Node documentation generator

Regenerates the per-category node reference pages under [`docs/nodes/`](../../docs/nodes/)
from the **compiled** shader node files in `src/nodes/shader/`.

The node `.py` files are auto-generated by the NodeCompiler, so socket names,
types, default values, ranges and panels in the docs are extracted straight from
that source — they cannot drift from the actual nodes. Only the prose (category
overviews, use cases, usage steps, and per-node one-line descriptions) is
hand-authored, in `doc_content.py`.

## Usage

No `bpy` required — the scripts parse the generated Python with `ast`. Any system
Python 3.10+ works (the dev venv is not needed):

```bash
python tools/nodedocs/extract_nodes.py   # src/nodes/shader/** -> nodes_data.json
python tools/nodedocs/gen_node_docs.py    # nodes_data.json    -> docs/nodes/*.md
```

`nodes_data.json` is an intermediate build artifact and is git-ignored.

## Files

| File | Role |
|------|------|
| `extract_nodes.py` | AST-parses every compiled node, dumps socket/panel ground truth to `nodes_data.json`. |
| `doc_content.py` | Authored prose: `CATEGORY_META` (per page) and `NODE_DESC` (per node). Edit this to improve descriptions. |
| `gen_node_docs.py` | Merges the two into `docs/nodes/*.md`. Mirrors the menu routing in `src/nodes/node_info.py`. |

## When to re-run

After recompiling nodes (`lspotato.compile_node_groups`) or otherwise changing
anything under `src/nodes/shader/`, re-run both scripts so the docs match.
To improve wording, edit `doc_content.py` and re-run `gen_node_docs.py` only.
