# LSPotato — Node System Technical Reference

**Version 2.0.0 | For add-on developers and contributors**

---

## 1. Overview

The node system in `src/nodes/` provides:

1. **A runtime framework** (`node.py`, `node_impl.py`, `node_info.py`) that discovers, registers, and exposes compiled shader nodes via Blender's Add Shader menu.
2. **A compiled shader library** (`src/nodes/shader/lscherry/`) of 230+ auto-generated Python node classes representing LSCherry toon-shader groups.
3. **A geometry node loader** (`src/nodes/geometry/`) that appends shipped node groups from a `.blend` library on file open.

Node source files in `src/nodes/shader/` are **auto-generated by the NodeCompiler** (see `src/features/node_compiler/`). Do not edit them manually — regenerate from the source `.blend` file.

## 3. Core Framework

### 3.1 `node.py` — Base Classes and Registry

**File:** [src/nodes/node.py](../src/nodes/node.py)

#### Class hierarchy

```
bpy.types.ShaderNodeCustomGroup
    └── ShaderNode(Node, bpy.types.ShaderNodeCustomGroup)
            └── ShaderNodeCompiled_<Name>   <- every compiled shader node

bpy.types.GeometryNodeCustomGroup
    └── GeometryNode(Node, bpy.types.GeometryNodeCustomGroup)
```

`Node` is a plain Python mixin (no `bpy.types` in its MRO) that provides all programmatic tree-building helpers.

#### Class-level registry

Two module-level dicts are populated at addon registration time:

| Dict | Key | Value |
|---|---|---|
| `_NODE_CLASS_REGISTRY` | `cls._PREFIX + cls.bl_label` (stable tree key) | Compiled class object |
| `_NODE_CLASS_BY_IDNAME` | `cls.bl_idname` | Compiled class object |

Public API:

```python
register_node_class(cls)          # called from src/__init__.py for every compiled class
clear_node_registry()             # called on unregister
get_node_class_by_idname(idname)  # returns class or None
ensure_node_group(key)            # returns or builds the bpy node group for a stable key
```

#### Stable key

The stable key used to look up and name a node group in `bpy.data.node_groups` is:

```python
key = cls._PREFIX + cls.bl_label
# e.g.  ".lscherry.core.ToonCore"
```

`_PREFIX = "."` for both `ShaderNode` and `GeometryNode`. Blender clamps ID names to 63 characters; the stable key is truncated silently by Blender if it exceeds this limit.

#### `ensure_node_group(key)`

Used by generated `createNodetree()` code when embedding a nested node group:

```python
ng = ensure_node_group(".lscherry.utils.bnodes.SomeUtil")
# Returns the existing bpy node group, or builds it from the registered class,
# or returns None if the key is unknown.
```

This replaces the fragile `bpy.data.node_groups.get(name)` pattern that fails on first-add.

#### `Node` mixin — tree-building helpers

| Method | Signature | Purpose |
|---|---|---|
| `createNodetree` | `(self, name)` | Override in each compiled class to build the node tree |
| `getNodetree` | `(self, name)` | Called from `init()` — binds the shared node tree to `self.node_tree` |
| `create_node_group` | classmethod | Builds the tree if absent; used by nested references and the Add operator |
| `addSocket` | `(is_output, sockettype, name)` | Creates an interface socket |
| `addNode` | `(nodetype, attrs)` | Creates an internal node with attribute dict |
| `getNode` | `(nodename)` | Finds an internal node by name |
| `innerLink` | `(socketin, socketout)` | Links two internal sockets via `path_resolve` |
| `value_set` | `(obj, path, value)` | Sets a dotted attribute path on a node |
| `free` | `(self)` | Removes the shared node group when the last user is deleted |

#### `create_node_group` — classmethod detail

Uses a `types.SimpleNamespace` proxy to drive `createNodetree()` without a live Blender node instance:

```python
proxy = types.SimpleNamespace(
    _PREFIX=cls._PREFIX,
    bl_label=cls.bl_label,
    node_tree=None,
    valuesUpdate=lambda *a, **k: None,
)
cls.createNodetree(proxy, key)
return proxy.node_tree or bpy.data.node_groups.get(key)
```

The proxy's `valuesUpdate` no-op prevents generated code that calls `self.valuesUpdate(None)` at the end of `init()` from raising when called outside a real node context.

#### `load_packaged_image(filename)`

Loads a texture from `src/nodes/shader/images/<filename>` with `check_existing=True` so multiple nodes referencing the same image share one datablock. Returns `None` gracefully on a missing file.

### 3.2 `node_impl.py` — Node Discovery (`NodeLib`)

**File:** [src/nodes/node_impl.py](../src/nodes/node_impl.py)

`NodeLib` recursively scans `src/nodes/shader/` at registration time and returns every `ShaderNodeCustomGroup` subclass found, ready to register with Blender.

#### `NodeLib.get_node_classes() -> list`

Safe entry point — never raises. Returns the full list of compiled classes.

#### Internal scan flow

1. `_scan_all()` walks `_shader_DIR.rglob("*.py")` in sorted order, skipping files whose stem is in `SKIP_STEMS = {"__init__", "node", "utils", "node_impl", "node_info"}`.
2. For each file, `_load_file(py_file)` computes the full dotted module name relative to the addon package root (e.g., `LSPotato.nodes.shader.lscherry.core.toon_core`) and loads it with `importlib.util.spec_from_file_location`.
3. All attributes of the loaded module are inspected; those that are:
   - a `type`
   - a subclass of `bpy.types.ShaderNodeCustomGroup`
   - not the base type itself
   - have `bl_label` and `bl_idname` in `__dict__`

   ...are collected and returned.
4. Duplicate class names are skipped (first occurrence wins).

> `GeometryNodeCustomGroup` subclasses are intentionally excluded — geometry nodes are loaded via the binary library, not registered as custom group types.

### 3.3 `node_info.py` — Menu Registration and Add Operator

**File:** [src/nodes/node_info.py](../src/nodes/node_info.py)

#### Category mapping

`_CATEGORY_MAP` is an ordered list of `(prefix, menu_path)` tuples. The first prefix that matches a node's `bl_label.lower()` determines its menu path. Order is most-specific first, most-generic last.

```python
_CATEGORY_MAP = [
    ("lscherry.external.michos.honkai_impact_3.", "LSCherry/External/Michos/Honkai Impact 3"),
    ("lscherry.external.michos.genshin_impact.",  "LSCherry/External/Michos/Genshin Impact"),
    ...
    ("lscherry.",                                 "LSCherry"),  # fallback
]
```

`_get_category(bl_label)` returns the matching path string. `_display_name(bl_label)` returns only the last `.`-separated segment for use as the menu item label.

#### `LSPOTATO_OT_add_lscherry_node` operator

`bl_idname = "lspotato.add_lscherry_node"`

All LSCherry menu items route through this operator instead of the stock `node.add_node`. Its `invoke()`:

1. Looks up the class by `node_idname` via `get_node_class_by_idname`.
2. Calls `cls.create_node_group()` to pre-build the entire tree (and all its nested dependencies) before Blender's `init()` runs.
3. Delegates to `bpy.ops.node.add_node("INVOKE_DEFAULT", ...)` so the node follows the cursor as usual.

This avoids the first-add "Missing Data Block" desync that occurs when deeply-nested nodes are built during `init()` inside the stock operator.

#### `_build_menu_classes(node_classes)`

Builds a complete hierarchy of `bpy.types.Menu` subclasses dynamically at registration time:

1. **Pass 0:** Groups nodes by category path.
2. **Pass 1:** Collects all ancestor paths so intermediate menus (e.g., `LSCherry/External/Michos`) are created even if they contain only sub-menus.
3. **Pass 2:** Builds a parent-to-direct-children map.
4. **Pass 3:** Creates one `type(...)` menu class per path. Each menu's `draw()` renders child sub-menus first, then a separator, then the direct node items.

Menu IDs are stable strings derived from the path:
```
LSCherry/External/Michos  ->  LSPOTATO_MT_LSCHERRY_EXTERNAL_MICHOS
```

The root menu ID is the constant `LSPOTATO_MT_LSCHERRY_ROOT`.

#### Public registration API

```python
ng_register(node_classes)   # call AFTER registering node classes
ng_unregister()             # call BEFORE unregistering node classes
```

`ng_register` also calls `bpy.types.NODE_MT_add.append(_add_to_shader_add_menu)` to inject the root `LSCherry` entry into the standard Add Shader menu.

#### `restore_undefined_nodes` — `load_post` handler

Scans all materials and node groups on file open. For every `NodeUndefined` node whose `type` attribute matches a known `bl_idname`:

1. Creates a replacement node of the correct type at the same location.
2. Copies default values and reconnects all links.
3. Removes the old `NodeUndefined` node.

Registration:

```python
register_restore_handler()    # appends to bpy.app.handlers.load_post
unregister_restore_handler()  # removes from load_post
```

## 4. Geometry Node System

### 4.1 `geometry/hashing.py` — Deterministic Node Tree Hashing

**File:** [src/nodes/geometry/hashing.py](../src/nodes/geometry/hashing.py)

Provides a stable MD5 hash of a node tree's content so the loader can decide whether to overwrite an existing group.

#### Key functions

| Function | Purpose |
|---|---|
| `to_serializable(val)` | Coerces socket `default_value` to a JSON-safe type |
| `_node_location(node)` | Returns node location rounded to 2 decimals |
| `_scalar_props(node)` | Collects content-bearing scalar props via introspection; excludes volatile UI attributes (`select`, `dimensions`, `width`, `height`, `location_absolute`) |
| `serialize_node_tree(nt)` | Returns a JSON-serializable dict: nodes sorted by name, links sorted by endpoint descriptions |
| `hash_node_tree(nt)` | Returns `hashlib.md5(json.dumps(serialize_node_tree(nt)).encode()).hexdigest()` |

This module has **no `bpy` imports at module level** — it is safe to unit-test without a running Blender instance.

The NodeCompiler writes a `hashes.json` file alongside `library.blend` at compile time using the same `hash_node_tree` function. The runtime loader reads `hashes.json` to compare against the live file.

---

### 4.2 `geometry/loader.py` — Runtime Geometry Node Loader

**File:** [src/nodes/geometry/loader.py](../src/nodes/geometry/loader.py)

Ensures the shipped geometry node groups exist in `bpy.data.node_groups` at the correct version, appending or overwriting from `library.blend` as needed.

#### `init_geometry_nodes()`

Main entry point. Loads `hashes.json`, collects the names of all shipped groups, queries which are absent or hash-mismatched, and calls `_bring_in()` for those groups.

#### `_bring_in(names, old_blocks, hashes)`

1. Appends the named groups from `library.blend` using `bpy.ops.wm.append(...)`.
2. For each name:
   - **Absent:** the appended block is the canonical version, used as-is.
   - **Hash mismatch (stale):** calls `old_block.user_remap(new_block)` to redirect all modifier/node references to the new version, then removes the old block.
   - **Hash match:** removes the newly appended duplicate, leaving the user's version untouched.

#### `geometry_load_post(dummy)` — `load_post` handler

Calls `init_geometry_nodes()` on every file open. Silent on errors — never raises.

Guard conditions: skips silently if `library.blend` or `hashes.json` is missing (e.g., development build without a compiled library).

#### Registration

```python
register_geometry_handler()    # appends load_post handler + runs init once
unregister_geometry_handler()  # removes load_post handler
```

## 5. Compiled Node File Format

Every file in `src/nodes/shader/` follows this template (auto-generated — do not edit):

```python
# AUTO-GENERATED by LSPotato NodeCompiler. Do not edit manually.
import bpy  # type: ignore
from ...node import ShaderNode, ensure_node_group

class ShaderNodeCompiled_Name(ShaderNode):
    bl_idname = 'ShaderNodeCompiled_Name'
    bl_label  = 'lscherry.<path>.Name'

    def init(self, context):
        self.getNodetree(self.bl_label)
        # default value assignments for output/input sockets
        ...

    def createNodetree(self, name):
        self.node_tree = bpy.data.node_groups.new(name, 'ShaderNodeTree')
        self.node_tree.color_tag = 'COLOR'

        # Interface sockets
        self.addSocket(False, 'NodeSocketColor', 'Base Color')
        self.addSocket(True,  'NodeSocketShader', 'Shader')

        # Internal nodes
        node_0 = self.addNode('ShaderNodeMixRGB', {
            'name': 'Mix',
            'blend_type': 'MIX',
            'inputs[0].default_value': 0.5,
        })

        # Nested group reference
        ng = ensure_node_group('.lscherry.utils.bnodes.SomeUtil')
        node_1 = self.addNode('ShaderNodeGroup', {'name': 'SomeUtil', 'node_tree': ng})

        # Links
        self.innerLink('nodes["Mix"].outputs[0]', 'nodes["SomeUtil"].inputs[0]')
        self.innerLink('nodes["SomeUtil"].outputs[0]',
                       'interface.items_tree["Shader"].identifier')
```

### Naming convention

`bl_label` encodes the full menu path using dot notation:

| `bl_label` | Blender menu path |
|---|---|
| `lscherry.PBR` | LSCherry |
| `lscherry.combiner.BlendColor` | LSCherry/Combiner |
| `lscherry.core.ToonCore` | LSCherry/Core |
| `lscherry.utils.bnodes.ColorMath` | LSCherry/Utils/BNodes |
| `lscherry.external.michos.genshin_impact.GI_BodyColor` | LSCherry/External/Michos/Genshin Impact |
| `lscherry.vfx.HologramShader` | LSCherry/VFX |

The `bl_idname` is always `ShaderNodeCompiled_<CamelCaseName>` and must be unique across all registered nodes.

## 6. Registration Sequence

The full registration order is enforced in `src/__init__.py`:

```
1.  NodeLib.get_node_classes()          # discover all compiled classes
2.  for cls in node_classes:
        bpy.utils.register_class(cls)
        register_node_class(cls)        # populate both registry dicts
3.  ng_register(node_classes)           # build and register menus + Add operator
4.  register_restore_handler()          # install NodeUndefined load_post handler
5.  register_geometry_handler()         # install geometry library load_post handler
```

Unregistration runs in reverse:

```
1.  unregister_geometry_handler()
2.  unregister_restore_handler()
3.  ng_unregister()                     # remove menus + Add operator
4.  for cls in reversed(node_classes):
        bpy.utils.unregister_class(cls)
5.  clear_node_registry()
```

## 7. Adding a New Compiled Node

Compiled nodes are generated, not written by hand. The intended workflow is:

1. Build or modify the node group in a `.blend` file.
2. Use the **NodeCompiler** operator (`lspotato.compile_node_groups`) in Blender to export Python source files to a `compiled/` folder.
3. Copy the generated files into the appropriate `src/nodes/shader/lscherry/<category>/` directory.
4. Run `potato reload` to reinstall and restart Blender.

If you must add a node manually (e.g., a simple wrapper), follow the compiled node template in section 5 and ensure:

- `bl_label` follows the dotted-path naming convention so the menu system places it correctly.
- `bl_idname` is unique and follows the `ShaderNodeCompiled_<Name>` pattern.
- `createNodetree` creates the node tree using the `name` argument passed by `create_node_group`.
- Any nested group is resolved via `ensure_node_group(key)`, not by a bare `bpy.data.node_groups.get(name)`.

## 8. Adding a New Category

To add a new top-level or nested category to the Add Shader menu:

1. Add a new entry to `_CATEGORY_MAP` in [src/nodes/node_info.py](../src/nodes/node_info.py) — **before** the `("lscherry.", "LSCherry")` fallback line and **after** any more-specific prefix it would shadow.
2. Update the `_ROUTES` table in `src/features/node_compiler/compiler/router.py` with the matching folder path.
3. Place compiled node files in the matching subfolder under `src/nodes/shader/lscherry/`.

The menu system builds the full hierarchy automatically from the category paths — no further registration code is needed.

## 9. Developer Notes

### Node tree sharing

All instances of the same compiled node class share one `bpy.data.node_groups` datablock, identified by the stable key. This is Blender's standard node-group sharing model. Editing one instance's internals in the node editor affects all instances.

### `valuesUpdate` pattern

Generated `init()` methods sometimes end with `self.valuesUpdate(None)`. This method is not defined on the base class — it is expected to be defined by the subclass for nodes that need to update socket visibility based on property values. For nodes that do not define it, `create_node_group()` provides a no-op via the `SimpleNamespace` proxy.

### Blender 5.x interface API

Compiled nodes use the Blender 5.x interface API for socket creation:

```python
self.node_tree.interface.new_socket(name, in_out="INPUT", socket_type="NodeSocketColor")
```

This API is not compatible with Blender 4.x or earlier. The add-on requires Blender 5.0 minimum (`bl_info["blender"] = (5, 0, 0)`).

### bpy-free unit testing

`geometry/hashing.py` is intentionally free of top-level `bpy` imports and can be tested with Python alone. `NodeLib._load_file` and `NodeLib._scan_all` require `bpy` at load time because the compiled files import it. Use `src/mock/` fixtures for unit testing node scanning without a running Blender instance.
